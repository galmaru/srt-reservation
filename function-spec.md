## 2. F-01 로그인 / 로그아웃

### 2.1 기능 설명

SRT 또는 KTX 계정으로 인증한다. 로그인 화면 상단에서 provider를 선택한다.

### 2.2 Provider 선택 UI

로그인 화면 상단에 **SRT / KTX 토글 탭**:
- SRT: 파란 색상
- KTX: 빨간 색상 + "예매 후 코레일 앱에서 결제 필요" 경고 문구

### 2.3 입력 항목

| 항목 | 타입 | 유효성 |
|------|------|--------|
| provider | hidden | "srt" 또는 "ktx" |
| 아이디 | text | 필수 |
| 비밀번호 | password | 필수 |

### 2.4 처리 흐름

```
provider 선택 + 자격증명 입력
  → provider == "srt" → srt_bot.login()
  → provider == "ktx" → korail_bot.login()
    ├─ 성공 → session(train_id / train_pw / provider) 저장 → /search
    └─ 실패 → 에러 메시지 표시
```

### 2.5 세션 구조

| 키 | 값 |
|----|----|
| `train_id` | 로그인 아이디 |
| `train_pw` | 로그인 비밀번호 |
| `provider` | "srt" \| "ktx" |

### 2.6 내비게이션

- 헤더: `SRT`(파랑) / `KTX`(빨강) provider 배지 표시
- **카드 설정** 메뉴: SRT 로그인 시만 노출

### 3.2 검색 조건

| 항목 | SRT | KTX |
|------|-----|-----|

| 날짜 | 필수 | 필수 |
| 출발 시각 이후 | 기본 08:00 | 기본 08:00 |

### 3.3 KTXTrainAdapter — 통합 인터페이스

korail2 `Train` 객체를 래핑하여 SRT Train과 동일한 속성/메서드를 노출한다.

| 속성/메서드 | KTX 매핑 |
|-------------|---------|
| `train_name` | `train.train_type_name` |
| `dep_station_name` | `train.dep_name` |
| `arr_station_name` | `train.arr_name` |
| `dep_time` (HHMMSS) | `train.dep_time` |
| `arr_time` (HHMMSS) | `train.arr_time` |
| `running_time` | `train.run_time` |
| `general_seat_available()` | `train.has_general_seat()` |
| `special_seat_available()` | `train.has_special_seat()` |
| `seat_available()` | general \|\| special |
| `waiting_available()` | 예약대기 가능 여부 (라이브러리 지원 시) |

| ⚡ 자동 예매하기 | 빈좌석 즉시 예매 또는 백그라운드 모니터 등록 |

## 4. F-03 예매 실행

### 4.1 기능 설명

선택 열차를 예매한다. 잔여석 여부·provider에 따라 분기한다.

### 4.2 처리 분기

```
잔여석 확인 (재검색)
  │
  ├─ 빈좌석 있음 → 즉시 예매
  │    ├─ 성공 → [SRT+카드] 자동결제 또는 [KTX] 앱 결제 안내
  │    │         → Slack 알림 전송 → 결과 화면
  │    └─ 실패 → 에러 메시지
  │
  └─ 빈좌석 없음 → 백그라운드 모니터 등록 → /monitors
```

### 4.3 예매 결과 화면 — provider별

| 케이스 | 표시 |
|--------|------|
| SRT + 자동결제 성공 | "예매 & 결제 완료" + Slack 전송 완료 표시 |
| SRT + 자동결제 실패 | 예약번호 + 결제 실패 사유 + "직접 결제" 버튼 |
| SRT + 카드 미저장 | 예약번호 + "결제하기" / "카드 저장하기" 버튼 |
| KTX | 예약번호 + "코레일 앱에서 20분 내 결제" 안내 |
| 예매 실패 | 에러 메시지 |

## 5. F-04 백그라운드 자동 예매

### 5.1 기능 설명

빈좌석이 없는 열차를 대상으로 **예매 → 예약대기 → 재시도** 순서로 자동 선점한다. 성공 시 Slack으로 알림을 전송한다.

> **삭제**: 빈좌석 알림 모드(`notify`) 제거. 단일 자동 예매 모드만 운영.

### 5.2 모니터 등록 정보

| 항목 | 설명 |
|------|------|
| `provider` | "srt" / "ktx" |
| `train_id` / `train_pw` | 폴링 시 재로그인용 |
| 열차 식별자 | `dep_time` (HHMMSS) |
| 예매 옵션 | `adult_count` / `seat_type` |
| `interval` | 폴링 간격 (초) |
| 카드 정보 | SRT + 카드 저장 시만, KTX는 항상 None |
| `slack_webhook_url` | 세션에서 전달 (설정된 경우) |

### 5.3 폴링 로직 (핵심 변경)

```
while status == "running":
    재로그인 → 열차 재검색

    ┌─ 빈좌석 있음 (seat_available)
    │    → make_reservation() 시도
    │    ├─ 성공 → status="done", result_type="reservation"
    │    │         → [SRT+카드] pay_reservation()
    │    │         → slack_notify("예매 완료")
    │    │         → return
    │    └─ 실패 → 예약대기 시도 (아래)
    │
    ├─ 예약대기 가능 (waiting_available)
    │    → make_waiting_reservation() 시도
    │    ├─ 성공 → status="done", result_type="waiting"
    │    │         → slack_notify("예약대기 등록 완료")
    │    │         → return
    │    └─ 실패 → interval초 후 재시도
    │
    └─ 모두 불가 → "[N회] 잔여석 없음" 메시지 → interval초 후 재시도
```

### 5.4 상태 흐름

```
running
  → done   (예매 또는 예약대기 성공)
  → cancelled (사용자 수동 중지)
```

> **삭제**: `available` 상태 제거 (알림 모드 삭제에 따라)

### 5.5 모니터 카드 UI

- 헤더: `SRT`(파랑) / `KTX`(빨강) 배지
- done 상태 메시지:
  - 예매 성공: "예매 완료 (예약번호: XXXXX)"
  - 예약대기 성공: "예약대기 등록 완료 (예약번호: XXXXX)"
- KTX done: "결제하기" 버튼 없음, "코레일 앱에서 결제" 안내

### 5.6 API 엔드포인트

| 메서드 | URL | 설명 |
|--------|-----|------|
| GET | `/api/monitor/{id}/status` | 현재 상태 JSON |
| POST | `/api/monitor/{id}/cancel` | 모니터 중지 |
| POST | `/api/monitor/{id}/delete` | 모니터 삭제 |

## 6. F-05 Slack 알림

### 6.1 기능 설명

예매 또는 예약대기 성공 시 Slack Incoming Webhook으로 메시지를 전송한다.

### 6.2 설정

| 항목 | 설명 |
|------|------|
| `SLACK_WEBHOOK_URL` | 환경변수 또는 웹 UI 설정 화면에서 입력 |
| 미설정 시 | 알림 전송 없이 정상 동작 |

### 6.3 전송 시점

| 시점 | 메시지 내용 |
|------|-------------|
| 즉시 예매 성공 | 예약번호 / 노선 / 출발 시각 / 결제 여부 |
| 예약대기 등록 성공 | 예약번호 / 노선 / 출발 시각 / "대기 중" 표시 |
| 백그라운드 예매 성공 | 위 동일 + "자동 예매 완료" 표시 |
| 백그라운드 예약대기 성공 | 위 동일 + "자동 예약대기 등록" 표시 |

### 6.4 메시지 포맷 (예시)

```
🚄 SRT 자동 예매 완료
노선: 수서 → 부산
열차: SRT 313  출발: 09:00
예약번호: 20261234
결제: 완료 ✅
```

```
🚄 KTX 자동 예약대기 등록
노선: 서울 → 부산
열차: KTX 103  출발: 09:20
예약번호: K20261050
결제: 코레일 앱에서 진행 필요
```

### 6.5 구현 위치

`slack_notifier.py` 신규 파일:
```python
def notify(webhook_url: str, message: dict) -> bool:
    """Slack Incoming Webhook으로 메시지 전송. 실패 시 False 반환 (예매는 정상 처리됨)"""
```

- Slack 전송 실패는 예매 결과에 영향을 주지 않음 (best-effort)

## 12. 공통 규칙

### 12.1 반봇

| provider | 방식 |
|----------|------|
| SRT | 매 검색마다 `NetFunnelHelper` 재초기화 |
| KTX | korail2 기본 세션 |

### 12.2 중복 예매 방지

예매/예약대기 성공 직후 즉시 `status="done"` — provider 무관 공통 적용.

### 12.3 Slack 전송 실패 처리

Slack 전송 실패는 예매 결과에 영향 없음. 오류 로그만 기록.

### 12.4 보안

| 항목 | SRT | KTX |
|------|-----|-----|
| 카드 정보 | Flask 암호화 쿠키 | 해당 없음 |
| 세션 쿠키 | HttpOnly, SameSite=Lax | 동일 |
| 모니터 API 소유자 검증 | train_id 기준 | 동일 |
| Slack Webhook URL | 환경변수 또는 세션 저장 | 동일 |
