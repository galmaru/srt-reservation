## 1. 개요

### 1.1 제품 설명

SRT(수서고속철도) 티켓을 자동으로 예매해주는 봇. 사용자가 원하는 열차를 지정하면 빈좌석이 생기는 순간 자동으로 예매 및 결제까지 처리한다.

### 1.2 핵심 문제

SRT는 인기 노선·시간대의 표가 수 분 안에 매진된다. 사용자가 수동으로 새로고침을 반복해 빈좌석을 잡는 것은 현실적으로 불가능하다.

### 1.3 목표

- 매진 열차에 빈좌석이 발생하는 즉시 자동 선점
- 저장된 카드정보로 결제까지 자동화하여 사람 개입 최소화
- CLI와 웹 양쪽에서 동일한 핵심 기능 제공

## 4. 비기능 요구사항

### 4.1 안정성

- NetFunnel 대기열 우회: 매 검색마다 `NetFunnelHelper` 재초기화로 "Wrong Server ID" 에러 방지
- 백그라운드 스레드는 데몬 모드로 실행 — 앱 종료 시 자동 정리
- `_monitor_lock`으로 딕셔너리 동시 접근 보호

### 4.2 보안

- 카드정보는 Flask 암호화 쿠키(서버사이드 서명)에만 저장, DB·로그 미기록
- `SECRET_KEY` 환경변수 미설정 시 재시작마다 모든 세션 무효화됨 (Vercel 배포 시 필수 설정)
- 세션 쿠키: `HttpOnly=True`, `SameSite=Lax`

### 4.3 성능

- 폴링 기본 3초 간격 (사용자 조정 가능)
- 동시 다중 모니터 스레드 지원

### 4.4 배포

- Vercel 서버리스 배포 지원 (`vercel.json`)
- Python 3.12 (`.python-version`)

## 6. 주요 플로우

### 6.1 웹 — 즉시 예매

```
로그인 → 열차 검색 → 열차 선택 → 예매 시도
  ↓ 잔여석 있음 + 자동예매 모드
즉시 예매 → [카드 저장 시] 자동결제 → 결과 화면
```

### 6.2 웹 — 백그라운드 모니터링

```
로그인 → 열차 검색 → 매진 열차 선택 → 모니터 등록
  ↓ 백그라운드 스레드 실행 (3초 간격 폴링)
빈좌석 감지
  ├─ 자동예매 모드 → 예매 → [카드 저장 시] 자동결제 → done
  └─ 알림 모드 → available 상태로 전환 (사용자 직접 예매)
```

### 6.3 CLI — 자동 모드

```
.env 로드 → 로그인 → 열차 검색 → 첫 번째 열차 예매
  → [AUTO_PAY=true] 카드결제 → 완료
```

## 8. 제외 범위

- 회원가입 / 비밀번호 변경 기능 없음 (SRT 공식 앱에서 처리)
- 어린이·경로우대 승객 자동 선택 없음 (현재 어른만 지원)
- 푸시 알림 / 이메일 알림 없음 (모니터 페이지 폴링으로 확인)
- 모니터 상태 DB 영속화 없음 (서버 재시작 시 모든 모니터 초기화)
