## 1. 개요

### 1.1 제품 설명

SRT(수서고속철도)와 KTX(한국철도공사) 티켓을 자동으로 예매해주는 봇. 사용자가 원하는 열차를 지정하면 빈좌석 예매 → 예약대기 순으로 자동 선점하며, 성공 시 Slack으로 알림을 전송한다. SRT는 카드 자동결제까지 지원하며, KTX는 예매 후 코레일 앱에서 직접 결제한다.

### 1.2 핵심 문제

SRT·KTX 모두 인기 노선·시간대의 표가 수 분 안에 매진된다. 사용자가 수동으로 새로고침을 반복해 빈좌석을 잡는 것은 현실적으로 불가능하다.

### 1.3 목표

- 빈좌석 발생 시 즉시 예매, 예약대기 가능 시 예약대기, 둘 다 불가 시 가능해질 때까지 재시도
- 예매/예약대기 성공 시 Slack 메시지 자동 전송
- SRT: 저장된 카드정보로 결제까지 자동화
- KTX: 예매 선점 후 코레일 앱 결제 안내
- CLI와 웹 양쪽에서 동일한 핵심 기능 제공

## 4. 비기능 요구사항

### 4.1 안정성

- **SRT**: NetFunnel 우회 — 매 검색마다 `NetFunnelHelper` 재초기화
- **KTX**: korail2 기본 세션 방식 사용
- 백그라운드 스레드: 데몬 모드, `_monitor_lock`으로 스레드 안전 보장
- `make_reservation()` 성공 즉시 `status="done"` 처리 (중복 예매 방지)

### 4.2 보안

- 카드정보(SRT 전용): Flask 암호화 쿠키에만 저장, DB·로그 미기록
- `SECRET_KEY` 환경변수 필수 (Vercel 배포 시)
- 세션 쿠키: `HttpOnly=True`, `SameSite=Lax`

### 4.3 성능

- 폴링 기본 3초 간격 (사용자 조정 가능)
- 동시 다중 모니터 스레드 지원

### 4.4 배포

- Vercel 서버리스 배포 지원 (`vercel.json`)
- Python 3.12

## 6. 주요 플로우

### 6.1 SRT — 즉시 예매 + 자동결제

```
로그인(SRT) → 열차 검색 → 잔여석 있음
  → 즉시 예매 → [카드 저장 시] 자동결제 → 결과 화면 → Slack 알림
```

### 6.2 KTX — 즉시 예매 + 수동결제 안내

```
로그인(KTX) → 열차 검색 → 잔여석 있음
  → 즉시 예매 → "코레일 앱에서 20분 내 결제하세요" 안내 → Slack 알림
```

### 6.3 공통 — 백그라운드 자동 예매

```
열차 선택 → 모니터 등록 (provider 기록)
  ↓ 백그라운드 폴링 (interval초 간격)
  ├─ 빈좌석 있음 → 즉시 예매 시도
  │    ├─ 성공 → [SRT+카드] 자동결제 → done → Slack 알림
  │    └─ 실패 → 예약대기 시도 (아래로)
  ├─ 빈좌석 없음 + 예약대기 가능 → 예약대기 등록
  │    └─ 성공 → done → Slack 알림
  └─ 빈좌석 없음 + 예약대기 불가 → interval초 후 재시도
```
